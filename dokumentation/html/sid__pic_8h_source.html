231 /*
232  * The note number from the note table is added to the duration above.
233  Then each note can be entered using only one number which is decoded by
234  your program. This is only one method of coding note values. You may be
235  able to come up with one with which you are more comfortable. The formula
236  used here for encoding a note is as follows:
237 
238  1) The duration (number of 1/16ths of a measure) is multiplied by 8.
239  2) The result of step 1 is added to the octave you've chosen (0-7).
240  3) The result of step 2 is then multiplied by 16.
241  4) Add your note choice (0-11) to the result of the operation in step
242  3.
243 
244  In other words:
245 
246  ((((D*8)+O)*16)+N)
247 
248  Where D = duration, O = octave, and N = note
249  A silence is obtained by using the negative of the duration number
250  (number of 1/16ths of a measure * 128).
251  */

351 /*
352  * Envelope Generator
353  *
354  * The volume of a musical tone changes from the moment you first hear it,
355  * all the way through until it dies out and you can't hear it anymore. When
356  * a note is first struck, it rises from zero volume to its peak volume. The
357  * rate at which this happens is called the ATTACK. Then, it fails from the
358  * peak to some middle-ranged volume. The rate at which the fall of the note
359  * occurs is called the DECAY. The mid-ranged volume itself is called the
360  * SUSTAIN level. And finally, when the note stops playing, it fails from
361  * the SUSTAIN level to zero volume. The rate at which it fails is called
362  * the RELEASE. Here is a sketch of the four phases of a note:
363  *
364  * +
365  * /|\
366  * / | \
367  * / | \
368  * SUSTAIN LEVEL . ./. .|. .+--------+
369  * / | | |\
370  * / | | | \
371  * / | | | \
372  * | | | | |
373  * | A | D | S | R |
374  *
375  * The parameters ATTACK/DECAY/SUSTAIN/RELEASE and collectively called ADSR
376  *
377  */

436  * FILTERING
437  *
438  * The harmonic content of a waveform can be changed by using a filter.
439  * The SID chip is equipped with three types of filtering. They can be used
440  * separately or in combination with one another. Let's go back to the
441  * sample program you've been using to play with a simple example that uses
442  * a filter. There are several filter controls to set.
443  * You add line 15 in the program to set the cutoff frequency of the
444  * filter. The cutoff frequency is the reference point for the filter. You
445  * SET the high and low frequency cutoff points in registers 21 and 22. To
446  * turn ON the filter for voice 1, POKE register 23.
447  * Next change line 30 to show that a high-pass filter will be used (see
448  * the SID register map).
449  *
450  * Try RUNning the program now. Notice the lower tones have had their
451  volume cut down. It makes the overall quality of the note sound tinny.
452  This is because you are using a high-pass filter which attenuates (cuts
453  down the level of) frequencies below the specified cutoff frequency.
454  There are three types of filters in your Commodore computer's SID chip.
455  We have been using the high-pass filter. It will pass all the frequencies
456  at or above the cutoff, while attenuating the frequencies below the
457  cutoff.

532 /*
533  * ADVANCED TECHNIQUES
534  *
535  * The SID chip's parameters can be changed dynamically during a note or
536  * sound to create many interesting and fun effects. In order to make this
537  * easy to do, digitized outputs from oscillator three and envelope
538  * generator three are available for you in registers 27 and 28, respec-
539  * tively.
540  * The output of oscillator 3 (register 27) is directly related to the
541  * waveform selected. If you choose the sawtooth waveform of oscillator 3,
542  * this register will present a series of numbers incremented (increased
543  * step by step) from 0 to 255 at a rate determined by the frequency of
544  * oscillator 3. If you choose the triangle waveform, the output will incre-
545  * ment from 0 up to 255, then decrement (decrease step by step) back down
546  * to 0. If you choose the pulse wave, the output will jump back-and-forth
547  * between 0 and 255. Finally, choosing the noise waveform will give you a
548  * series of random numbers. When oscillator 3 is used for modulation, you
549  * usually do NOT want to hear its output. Setting bit 7 of register 24
550  * turns the audio output of voice 3 off. Register 27 always reflects the
551  * changing output of the oscillator and is not affected in any way by the
552  * envelope (ADSR) generator.
553  *
554  * Register 25 gives you access to the output of the envelope generator
555  * of oscillator 3. It functions in much the same fashion that the output of
556  * oscillator 3 does. The oscillator must be turned on to produce any output
557  * from this register.
558  * Vibrato (a rapid variation in frequency) can be achieved by adding the
559  * output of oscillator 3 to the frequency of another oscillator. Example
560  * Program 6 illustrates this idea.
561  */
