◆ copyPoweronPattern()

void libsidplayfp::copyPoweronPattern ( sidmemory &  mem)

Copy in power on settings. These were created by running the kernel reset routine and storing the useful values from $0000-$03ff. Format is:

  • offset byte (bit 7 indicates presence rle byte)
  • rle count byte (bit 7 indicates compression used)
  • data (single byte) or quantity represented by uncompressed count all counts and offsets are 1 less than they should be

◆ magic

const uint8_t magic
Initial value:
=
0xff

Magic value for lxa and ane undocumented instructions. Magic may be EE, EF, FE or FF, but most emulators seem to use EE. Based on tests on a couple of chips at http://visual6502.org/wiki/index.php?title=6502_Opcode_8B_(XAA,_ANE) the value of magic for the MOS 6510 is FF. However the Lorentz test suite assumes this to be EE.
